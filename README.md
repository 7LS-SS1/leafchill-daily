# ระบบจัดการร้านอาหาร

เว็บแอพภาษาไทยสำหรับจัดการผู้ใช้งาน พนักงาน เวลาเข้า-ออกงาน กฎการมาสาย และการคำนวณค่าแรงรายวัน

## เทคโนโลยี

- Node.js HTTP server แบบไม่มี dependency ภายนอก
- Frontend: HTML, CSS, JavaScript
- Storage สำหรับตัวอย่าง: `data/database.json`
- โครงสร้างฐานข้อมูลสำหรับย้ายไป SQLite/PostgreSQL: `schema.sql`
- Password hash ด้วย `crypto.pbkdf2Sync`
- Session cookie แบบ `HttpOnly`

## วิธีรัน

Production/local ปกติ:

```bash
npm run seed
npm start
```

เปิดเว็บที่:

```text
http://localhost:3000
```

## วิธีรันเวอร์ชันทดสอบ

เวอร์ชันทดสอบใช้ฐานข้อมูลคนละไฟล์กับ production คือ `data/database.test.json` และรันที่ port `3100`

```bash
npm run seed:test
npm run start:test
```

เปิดเว็บทดสอบที่:

```text
http://localhost:3100
```

ในหน้าเว็บจะมีป้าย `TEST VERSION` เพื่อป้องกันการสับสนกับข้อมูลจริง

## บัญชีทดสอบ

```text
SYSTEM: system / system1234
Owner:  owner  / owner1234
Staff:  staff01 / staff1234
```

## ฟีเจอร์ที่มี

- Login / Logout
- จัดการผู้ใช้งาน: เพิ่ม แก้ไข ลบ username/password/role
- จัดการข้อมูลพนักงาน: เพิ่ม แก้ไข ลบ ค้นหา และดูข้อมูลในตาราง
- บันทึกเวลาเข้า-ออกงานรายวัน
- หน้า Staff สำหรับเช็คเข้างาน/ออกงานอัตโนมัติ พร้อมนาฬิกาเวลาประเทศไทยขนาดใหญ่
- แสดงชั่วโมงทำงานจริงแบบ realtime ตั้งแต่เวลาเข้างานจนถึงเวลาออกงาน
- ปฏิทินรายเดือนในหน้าบันทึกเวลาเพื่อดูรายการตามวันที่
- ตั้งค่ากฎการมาสายจากหน้าเว็บโดย role `SYSTEM` และ `Owner`
- ตั้งค่าค่าแรงรายวันของพนักงานโดย role `SYSTEM` และ `Owner`
- คำนวณค่าแรงสุทธิหลังหักค่ามาสายอัตโนมัติ
- เก็บประวัติการคำนวณย้อนหลังใน `wageCalculations`
- ค้นหารายงานตามช่วงวันที่และรายบุคคล
- Export รายงานค่าแรงเป็น CSV
- Audit log สำหรับการเปลี่ยนแปลงข้อมูลสำคัญ

## กฎเวลาเริ่มต้น

- ก่อนหรือเท่ากับ 16:30: มาปกติ
- หลัง 16:30 ถึงก่อน 17:00: หัก 20 บาท
- ตั้งแต่ 17:00 ถึงก่อน 18:00: หัก 50 บาท
- ตั้งแต่ 18:00 เป็นต้นไป: ไม่ได้รับค่าแรง
- หากออกงานก่อนเวลา 00:30 ของวันถัดไป: ถือว่าทำงานไม่ครบเวลา / ออกก่อนเวลา

กฎเหล่านี้แก้ไขได้จากเมนู `กฎเวลา` โดยไม่ต้องแก้โค้ด

## การออกงานอัตโนมัติ

ระบบใช้ timezone `Asia/Bangkok` สำหรับการแสดงและบันทึกเวลา หากหลังเวลา 06:00 ของวันใหม่แล้วยังมีรายการเข้างานที่ไม่มีเวลาออก ระบบจะปิดรายการนั้นอัตโนมัติด้วยเวลาออก `00:30` ของวันถัดไป เช่น เข้างานวันที่ `2026-03-01` เวลา `16:00` แล้วไม่เช็คออก เมื่อถึงหลัง `06:00` ของวันที่ `2026-03-02` ระบบจะบันทึก `checkOut` เป็น `00:30` และ `checkOutDate` เป็น `2026-03-02`

## หมายเหตุสำหรับต่อยอด Production

โปรเจกต์นี้เน้นให้รัน local ได้ทันทีและเห็น business logic ครบก่อน หากนำไปใช้จริงควรต่อยอดเป็น Next.js หรือ Express พร้อม Prisma + PostgreSQL/SQLite, เพิ่ม CSRF protection, HTTPS, migration, backup และ test suite เพิ่มเติม
